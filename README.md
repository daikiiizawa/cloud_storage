# クラウドストレージ

## 概要
* 100MBまで使える会員制のクラウドストレージ
* ドラッグ&ドロップでかんたんアップロード
* 1ファイル10MBまで形式制限なし

## 機能要件
* ファイルアップロード方式は「ドラッグ &ドロップ」と「ファイル選択」の両方が可能
* ディレクトリ作成機能は不要
* アップロードファイルの形式に制限は設けない
* ユーザー単位でサーバー側にディレクトリを切り、ファイルを保存する
* ユーザー情報は別途データベースに格納するものとする
* ユーザー登録はメールアドレスとパスワードを必須とする
* ユーザーログインには登録メールアドレスとパスワードを使用する
* 1ファイル上限 10MBまで
* 1ユーザー上限 100MBまで

## システム要件
* サーバーサイド言語は「 PHP5.6」
* フレームワークの使用は問わない
* ファイルアップロードのライブラリは「 Dropzone」を使用※
* サーバーOSは「CentOS6.7」
* データベースは「 MySQL5.6」
* その他のライブラリ等については自由に使用できる  
※PHPの通常のファイルアップロード機能も実装できると尚可

##  UI/UX要件(1)
* ユーザー登録画面、ログイン画面、ファイル一覧画面、ファイルアップロード画面を必須とする
* ファイルアップロード完了後は自動的に一覧画面に戻り、 画面内に「アップロード完了しました」と表示する
* ファイルの削除はファイル一覧画面から対象ファイルに チェックを入れることで可能とし、削除後は「削除が完了 しました」と表示する
* ログアウト画面はログイン画面と共用とし、ログアウト後に 「ログアウトしました」と表示する

##  UI/UX要件(2)
* 1ファイル 10MBの上限を超えた場合はエラー表示する
* 1ユーザー 100MBの上限を超えた場合、ファイルアップロード ボタンを無効とし、上限に達している旨を画面に表示する
* 各画面には共通ヘッダをもたせ「クラウドストレージ」という表記を入れる
* ベースデザインはフラットデザインとする
* 無駄の少ないシンプルなデザインが好ましい
* 基本フォントサイズは 14pxとする
* レスポンシブは非対応でよい
* cssファイル、 javascriptは外部ファイル読込みとする
